# LinglongReport
傻瓜式报表查询分析

针对无技术背景的业务人员，设计一个不仅是“报表生成器”，而且是一个“业务问答工作台”。
核心逻辑是：**傻瓜式查询，屏蔽技术参数，翻译业务意图。**

---

## 1. 交互中心：从“填表”转向“对话”

传统的报表工具需要选表、选字段、选过滤条件。我们的设计采用“搜索即报表”的逻辑。

### **A. 语义化搜索框**

用户只需输入一句话，系统自动识别报表类型、分析维度和度量指标的要素。

* **输入示例**：*“对比去年，看看上个月上海区域大类名称是‘生鲜’的销售额和毛利率趋势。”*
* **系统拆解**：
* **报表类型识别**：识别到“对比”和“趋势”，自动匹配【期间对比】+【期间趋势】。
* **维度匹配**：识别到“上海区域”（店铺信息-区域）、“生鲜”（分类信息-大类名称）。
* **指标匹配**：识别到“销售额”和“毛利率”（指标-销售类）。

### **B. 乐高式“意图卡片”**

如果用户不想打字，提供拖拽式的“业务积木”，将复杂的维度和指标卡片化：

* **维度堆叠**：通过店铺、商品、分类等彩色标签，让用户像搭积木一样选择。
* **指标自选**：将“销售、库存、进货”分类排列，点击即添加。

---

## 2. 预设场景：“傻瓜”模版

用户往往不知道该用什么格式，系统事先将各种常用格式封装为“分析场景”：

| 场景名称 | 业务语言 | 自动匹配逻辑 |
| --- | --- | --- |
| **查数助手** | “我就想看看昨天卖了多少” | 对应**期间查询**：固定日期，展示选定维度的指标汇总。 |
| **对比PK** | “跟上周比，到底涨了还是跌了” | 对应**期间对比**：自动生成环比、同比列，红绿箭头标出波动。 |
| **趋势观察** | “帮我看看这半个月的走势” | 对应**期间趋势**：自动生成折线图，按天展开指标。 |
| **时段爆点** | “一天里哪个点生意最好” | 对应**时段查询**：按24小时分布，识别高峰期。 |

---

## 3. 智能钻取：像剥洋葱一样分析

业务用户最怕“报表是死的”。可以利用分析维度中的层级关系，实现**自动钻取**。

* **逻辑层级**：区域 -> 督导 -> 店铺 -> 货架。
* **操作体验**：
1. 用户在报表中点击“华东大区”。
2. 报表不跳转，直接在下方展开该大区下的所有“区域名称”。
3. 点击某个区域，继续展开“店铺名称”。
4. 点击店铺，直接显示该店的“商品销量ABC级别”分布。

---

## 4. 结果呈现：不仅仅是表格

对于非技术用户，一堆数字是枯燥的。系统应提供“AI 结论前置”：

### **A. 自动图表选择**

* 选择“区域”+“销售额” -> 自动出**柱状图**。
* 选择“日期”+“毛利率” -> 自动出**折线图**。
* 选择“分类”+“库存金额” -> 自动出**饼图/环图**。

### **B. 业务警报器**

利用度量指标定义预警阈值（如“周转天数”、“动销率”），系统自动高亮异常：

* **库存预警**：如果“周转天数” > 目标值，单元格自动变红，并气泡提示：“该类商品积压严重，建议检查‘收货金额’”。
* **动销预警**：如果“动销率”低于20%，自动弹出“全店销量ABC级别”为C类的商品清单。

---

## 5. 灵活的“一键转文档/一键复制图片” 

可以增加一个“一键生成周报”功能：

1. 用户选定维度和指标。
2. 系统点击“生成报告”。
3. 利用 LLM 调用报表数据，自动写一段文字摘要：*“本月销售额环比增长10%，但生鲜大类的毛利率下降了5%，主要受进价波动影响，建议关注供应商XXX的采购协议。”*
4. 利用 AI 进行数据分析诊断，提供建议，生成诊断报告。

---

### **设计建议总结**

* **输入**：自然语言 + 标签云（维度/指标）。
* **中间层**：自动识别报表类型的格式逻辑。
* **输出**：可视化图表 + 异常高亮 + AI 业务摘要/AI 数据诊断。
